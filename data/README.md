# data/ — dynamic data (gitignored)

This folder holds all per-homework dynamic data. **Its contents are gitignored** (see
`.gitignore`) — only this README is tracked, so `git pull`/`push` never conflicts with
large submission files or generated PDFs.

## Expected layout (created by the tools)

```
data/
└── hw1/                         # one folder per homework (hw1, hw2, ...)
    ├── submissions/
    │   ├── _index.json          # all students (written by ingest_submissions.py)
    │   └── <student_id>/
    │       ├── _info.json       # file inventory + classification
    │       ├── _exam_input.md   # bundle for exam generation (prepare_exam_inputs.py)
    │       └── _processed/
    │           └── images/      # downscaled result-image thumbnails
    ├── exams/
    │   └── <student_id>/
    │       ├── exam.json        # generated by Claude (/generate-exam)
    │       ├── exam.pdf         # student copy (render_exam_pdf.py)
    │       └── key.md           # TA/AI grading key — do NOT distribute
    └── answer-sheets/
        └── answer_sheets_<code>.pdf
```

## Usage

Drop raw homework data anywhere (each subfolder = one student), then run:

```bash
python tools/ingest_submissions.py --input <raw-folder> --hw 1 \
    --output data/hw1/submissions --extract-zips
```

See the repo root `README.md` / `CLAUDE.md` for the full workflow.
